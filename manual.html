2.1 새 급여월 생성

매월 초, 새 급여월을 생성합니다. 이 한 번의 클릭으로 대부분의 데이터가 자동 생성됩니다.

1 대시보드에서 [새 월 생성] 버튼 클릭
우측 상단 파란색 버튼
2 급여월과 지급일자 입력
- 급여월: 해당 월 선택 (예: 2026-04)
- 지급일자: 급여 지급 예정일 (예: 2026-05-02)
3 [생성] 클릭
자동으로 처리되는 항목:
- 전월 직원 정보 및 참여율 복제
- 4대보험료 6종 자동 산출 (국민연금, 건강, 요양, 고용, 산재)
- 소득세/주민세 간이세액표 자동 조회
- 퇴직연금 자동 계산 (연봉/12)
- 38개 사업별 인건비 배분 자동 생성
- 사업별 지출 요약 자동 집계
소요 시간: 약 5~10초. 84명 x 38개 사업 기준으로 약 700건의 데이터가 자동 생성됩니다.

2.2 변경사항 입력

새 월 생성 후, 전월과 달라진 부분만 수정합니다.

변경이 필요한 경우:

변경 유형수정 방법빈도
참여율 변경Supabase 대시보드에서 project_assignments 테이블의 participation_rate 수정매월 일부
초과근무 시간초과근무 데이터 입력 후 재계산매월
중도 입사직원 추가 + 참여율/참여일수 설정발생시
중도 퇴사직원 비활성화 + 해당 월 참여일수 조정발생시
연봉 변경employees 테이블의 annual_salary 수정연 1회

2.3 자동 재계산

변경사항 입력 후, 보험료/소득세/배분을 일괄 재계산합니다.

1 대시보드에서 [자동 재계산] 버튼 클릭 (초록색)
확인 팝업에서 [확인] 클릭
2 재계산 완료 메시지 확인
재계산 대상:
- 개인공제 (보험료 + 소득세/주민세) 전원 재계산
- 기관부담 (보험료 + 퇴직연금) 전원 재계산
- 사업별 배분 (참여율 비례) 전체 재생성
- 사업별 지출 요약 재집계
주의: 재계산은 기존 공제/기관부담/배분 데이터를 삭제하고 새로 생성합니다. 수동으로 보정한 값이 있다면 재계산 후 다시 확인해주세요.

2.5 정산시스템별 내보내기

확정 후 정산 시스템에 제출할 데이터를 다운로드합니다.

1 대시보드에서 [내보내기] 버튼 클릭 (회색)
드롭다운 메뉴가 나타납니다
2 원하는 정산시스템 선택
- 전체 내보내기: 모든 사업의 지출내역
- E-나라도움: 자유무역, 군산의봄, 고용특구, 잡센터류 등 16개 사업
- RCMS: 김제 퀵스타트, 완주 퀵스타트 등 2개 사업
- 보탬e: 화학물질, 특수분야, 친환경선박 등 8개 사업
- 운영비 계좌(기업은행): 운영비, 스마트제조 등 5개 사업
- 계좌이체(기업은행): 김제일자리센터 1개 사업
3 CSV 파일 다운로드 확인
Excel에서 열어 정산시스템 제출 양식에 맞게 활용합니다.

4.2 소득세 간이세액표

국세청 근로소득 간이세액표를 시스템에 내장하여 자동 조회합니다.

계산 기준:

항목산출 방법
과세소득월급여(과세) + 직책수당 + 초과수당
부양가족수직원관리의 부양가족 컬럼 (본인 포함)
소득세간이세액표[과세소득 구간][부양가족수] x 세율(100% 또는 120%)
주민세소득세 x 10% (10원 미만 절사)

4.3 사업별 배분 로직

급여 배분:

사업별 급여 = 연봉월액 x 참여율 / 기준일수 x 참여일수

보험료/세금 배분:

사업별 보험료 = 전체 보험료 x 참여율 (비례배분)

반올림 잔액 보정:

비례배분 시 반올림으로 인한 +-1원 차이는 최대 참여율 사업에 자동 할당됩니다. 기존 엑셀에서 수동으로 +1/-1 보정하던 작업이 불필요합니다.

4.4 초과수당 계산식

항목계산식
통상시급연봉월액 / 209시간
지급단가통상시급 x 1.5배
초과수당MIN(지급단가 x 인정시간, 350,000원)
350,000원 상한: 초과수당은 월 350,000원을 초과할 수 없습니다. 상한 초과 시 자동으로 350,000원이 적용됩니다.

5. 자주 묻는 질문 (FAQ)

Q1. 새 직원이 입사했을 때는?

Supabase 대시보드에서 employees 테이블에 새 직원 추가 후, project_assignments에 참여율을 설정합니다. 그 후 [자동 재계산]을 실행하면 보험료/소득세/배분이 자동으로 생성됩니다.

Q2. 직원이 퇴사했을 때는?

employees 테이블에서 is_active를 false로 변경합니다. 해당 월의 참여일수를 실제 근무일수로 조정한 후 [자동 재계산]을 실행합니다.

Q3. 연봉이 인상되었을 때는?

employees 테이블의 annual_salary를 변경합니다. 다음 월 생성 시 자동으로 반영됩니다. 이미 생성된 월에 반영하려면 payroll_details의 monthly_salary도 수정한 후 [자동 재계산]을 실행합니다.

Q4. 보험료가 공단 고지서와 다를 때는?

시스템은 요율 기반으로 계산하므로, 공단의 정산/조정에 의해 수원~수십 원 차이가 발생할 수 있습니다. 이 경우 Supabase에서 해당 직원의 personal_deductions 또는 employer_contributions 값을 직접 수정하세요.

Q5. 참여율을 변경하고 싶을 때는?

Supabase 대시보드에서 project_assignments 테이블의 participation_rate를 수정한 후 [자동 재계산]을 실행합니다. 보험료/세금/배분이 새 참여율 기준으로 재계산됩니다.

Q6. 소득세가 120% 적용인 직원은?

employees 테이블의 tax_rate를 120으로 설정하면 소득세가 간이세액의 120%로 자동 계산됩니다. (예: 비거주자 또는 선택적 120% 원천징수)

Q7. 국민연금 면제자는 어떻게 처리하나요?

새 월 생성 시, 전월에 국민연금이 0원이었던 직원은 자동으로 면제 처리됩니다. 신규 면제자가 발생한 경우 Supabase에서 해당 직원의 personal_deductions에서 national_pension을 0으로 수정하세요.

Q8. 이전 월 데이터를 수정할 수 있나요?

대시보드 상단의 월 선택 드롭다운에서 해당 월을 선택한 후 [자동 재계산]을 실행하면 됩니다. 단, 이미 정산이 완료된 월은 수정 시 주의가 필요합니다.

Q9. CSV 파일을 엑셀에서 열 때 한글이 깨진다면?

CSV 파일은 UTF-8 BOM 형식으로 저장됩니다. Excel에서 바로 더블클릭으로 열면 정상 표시됩니다. 만약 깨지면: Excel > 데이터 > 텍스트에서 > 인코딩 UTF-8 선택
